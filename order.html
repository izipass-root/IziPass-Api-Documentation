Orders API

Last updated: February 26th, 2020

Get Started

 

  • Create a IziPass Agent account:

    A unique API access key will be generated automatically after registration and contract finalization.

    Public ApiKey - unique ID of your company in IziPass system

    Private ApiKey - a secret API access key (not available to anyone except your developer)

  • Make an API request according to the technical documentation.

The following options are required for JSON create orders model generation

 

Request parameters
serviceId Service ID. Required, GUID
flightNumber Customer’s flight number for the verification of service times, string. Required, Min length is 4, max length is 50
serviceDate Order date and time in local times of the respective airport. Required, datetime in ISO 8601 notation
adultCount Number of adults in the order. Required, integer
childCount Number of children in the order. Required, integer
infantCount Number of infants in the order. Required, integer
comment Additional information provided by the Customer. Optional, string
firstName Customer's first name. Required, string
lastName Customer's last name. Required, string
phone Customer's phone number in international format. Required, string
Validated according to regex
email Customer's email address. Required, string
Validated according to regex
otherPassengersContactDetails Info about other passengers in order, like name, phone, birthday date, etc. Required if passengers more than 1, string
culture Country/region and language code to set the language for voucher generation, string. Supported languages: en-US, ru-RU
expectedPrice Expected order price, calculated on your side, decimal. Passing this param means that you want to validate price if the service price was changed. In case of difference between expectedPrice and orderPrice API will return a validation error. Currency for this value depends on your account balance currency.

To perform a request for an order creation and payment please observe the following rules

To call the IziPass API you should pass the data and signature parameters via the POST method, where:

  • data - json string with APIs parameters encoded by the function base64, base64_encode( json_string ),

  • signature - is the unique signature of each request base64_encode( sha1( private_key + data + private_key) ),

  • base64_encode - returns a string encoded by the base64,

  • sha1 - the hash is returned as a binary string of 20 characters.

 

data and signature formation, example:

 

Data parameters
orders Order list of JSON create order model
publicApiKey Your Public ApiKey, GUID

 

Order requirements:

  • Each order should contain at list one adult passenger and not more than 50.

  • Two infants per each adult max.

  • A future date can only be selected as the date of Service and it must fall under the respective Service work timeframe (preOrderPeriodInMinutes, weekSchedule, unavailabilityPeriods).

  • A positive Agent account balance is required.
